SecureFill AI — Privacy Policy

Last updated: June 2026

Overview

SecureFill AI is a browser extension that helps you autofill non-sensitive form fields (name, email, phone, address, public profile links) locally in your browser. This policy explains what data is stored and how it is handled.

Data Collected

  • Profile fields you explicitly enter in the extension popup (e.g., name, email, phone, address). These are stored locally in chrome.storage.local.
  • Autofill action logs limited to domain, timestamp, number of fields filled and success flag. These logs are stored locally to provide a history within the extension.

What we do NOT store or collect

  • We never store or transmit passwords, payment details, credit card numbers, or other sensitive personal data.
  • We do not send profile data to any remote servers by default.

Permissions

The extension requests the following permissions:

  • storage — to save your profile and settings locally.
  • scripting and activeTab — to detect forms and inject autofill actions into active pages.
  • host_permissions — used to allow content scripts to run on pages you visit (declared in the manifest). We request sites access for form detection only.

Consent & Controls

Autofill is opt-in. You must save a profile and give consent before any autofill occurs. An optional setting enables autofill-on-load; this is off by default and must be enabled in the popup.

Third-party services

By default, SecureFill does not communicate with third-party servers. If you enable any optional integrations that require external services, we will disclose them clearly in the extension and listing.
